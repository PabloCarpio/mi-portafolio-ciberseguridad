El Escenario de Ataque (The Kill Chain)

Simulación de una intrusión en una aplicación web empresarial. Identificación de vectores de ataque mediante técnicas de reconocimiento pasivo y activo. Se exploraron puntos débiles en la lógica de negocio y la validación de entradas para escalar privilegios y obtener acceso a datos sensibles.

Hallazgos Técnicos

Cross-Site Scripting (XSS) Reflejado

Ejecución de scripts maliciosos en el lado del cliente para robo de cookies de sesión, permitiendo el secuestro de cuentas y la suplantación de identidad.

Inyección SQL (SQLi)

Bypass de autenticación mediante la manipulación de consultas SQL, logrando acceso no autorizado a la base de datos de usuarios y comprometiendo la integridad de los datos.

Resolución y Hardening (El Valor Añadido)

No solo atacamos, protegemos. Las siguientes medidas fueron implementadas para fortalecer la seguridad de la aplicación y el servidor:

  • Implementación de Content Security Policy (CSP) para mitigar la ejecución de scripts no autorizados.
  • Sanitización rigurosa de entradas en todos los formularios para prevenir inyecciones.
  • Configuración robusta de Firewalld y cierre de puertos críticos (SSH, RDP, etc.) en el servidor Linux.

Code Snippet: Ejemplo de Firewall Rules (AlmaLinux)

# Permitir tráfico web (HTTP/HTTPS)
sudo firewall-cmd --permanent --add-service=http
sudo firewall-cmd --permanent --add-service=https

# Permitir SSH (cambiar puerto si se ha modificado)
sudo firewall-cmd --permanent --add-port=22/tcp

# Bloquear todo lo demás por defecto (política implícita deny)
sudo firewall-cmd --set-default-zone=drop

# Recargar reglas
sudo firewall-cmd --reload

echo "Firewall configurado exitosamente."
